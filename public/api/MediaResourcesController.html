MediaResources

MediaResources are the core content of MAdeK. They are seperated in MediaEntries and MediaSets (Collection of MediaResources).

Get a collection of MediaResources

/media_resources [GET]

Optional Parameters

  • ids Array A collection of MediaResources you want to fetch informations for.
  • type String Filter the response by MediaResource types: "media_sets" | "media_entries".
  • sort String Sort the response (DESC) by: "updated_at"(Default) | "created_at" | "random".
  • query String Make a search request which searches in all MetaData of all MediaResources responding with matched MediaResources.
  • with[meta_data] Hash Adds MetaData to the responding collection of MediaResources and forwards the hash as options to the MetaData.
  • with[meta_data][meta_context_names] Array Adds all requested MetaContexts in the format: ["context_name1", "context_name2", ...] as MetaData to the responding MediaResources.
  • with[meta_data][meta_key_names] Array Adds all requested MetaKeys in the format: ["key_name1", "key_name2", ...] as MetaData to the responding MediaResources.
  • with[image] Hash Request the image of the MediaResources. You can define the responding image format like {"image": {"as": "base64" | "url"}}. The image size can be requested with {"image": {"size": "small"(100x100) | "small_125"(125x125) | "medium"(300x300) | "large"(620x500) | "x_large"(1024x768) }}
  • with[filename] Boolean Request the filename of the MediaResources.
  • with[media_type] Boolean Request the media_type of the MediaResources.
  • with[flags] Boolean Request status indicator informations (about permissions and favorites related to the current user) for the responding MediaResources.

Response Fields

  • id Integer The id of the MediaResource
  • meta_data Hash The MetaData of the MediaResource (To get a list of possible MetaData - or the schema - you have to consider the MetaDatum resource)
  • filename String The Filename of a MediaEntry's MediaFile (in case of MediaSets its null)
  • media_type String The Mediatype of a Media Resource (Video, Audio, Image or Doc;in case of MediaSets its Set)
  • is_public Boolean The is_public status.
  • is_private Boolean The is_private status.
  • is_shared Boolean The is_shared status.
  • is_editable Boolean The is_editable status.
  • is_managable Boolean The is_managable status.
  • is_favorite Boolean The is_fa status.
  • children Array The children of a MediaResource (only for MediaSets)..
  • parents Array The parents of a MediaResource.

Examples

Requesting the index of MediaResources without any attributes. Request media_resources.json Response 
{
  "media_resources": [
    {"id":1},{"id":2},{"id":3}],
  "pagination": {
    "total":3,
    "page":1,
    "per_page":36,
    "total_pages":1
  }
}
Responding with the index of media resources. Default sorting at on the update_at attribute latest first. You get 36 elements per page and informations about the current pagination.

Children are not responding for MediaEntries. The children uses a "nested" pagination for its own. If you want to controll the pagination inside the children pass a "pagination" attribute to the children value (e.g. {"pagination":{"page":2}}). If you want to forward a "with" to the children to request more nested informations just parse a Hash to the value containing the with informations (e.g. {"with":{"children":{with:{"media_type": true}}}}).

MediaEntries and MediaSets are responding with parents. The parents uses a "nested" pagination for its own. If you want to controll the pagination inside the parents pass a "pagination" attribute to the children value (e.g. {"pagination":{"page":2}}). If you want to forward a "with" to the parents to request more nested informations just parse a Hash to the value containing the with informations (e.g. {"with":{"parents":{"with":{"media_type": true}}}}).

Get the image of MediaResource:

/media_resources/:id/image [GET]

Required Parameters

  • id Integer The id of the MediaResource you want to fetch the image for.

Optional Parameters

  • size String Set the responding size of the image: "small"(100x100) | "small_125"(125x125) | "medium"(300x300) | "large"(620x500)[DEFAULT] | "x_large"(1024x768).
